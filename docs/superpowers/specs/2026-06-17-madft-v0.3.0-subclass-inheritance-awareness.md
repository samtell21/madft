# madft v0.3.0 — subclass-inheritance awareness

Status: approved design (pending spec review). Builds on v0.2.2.

## Problem

The freedesktop subclass DAG (`text/x-python → text/plain`, `application/x-ipynb+json →
application/json → …`) is currently a **read-only annotation** (`info`'s "inherits if
unset"). It drives nothing. Consequences the user hit:

- `madft ls Text.Development` shows almost nothing without `-a`, because the presence
  filter only counts **exact** `MimeType=` declarers — `text/x-python` is hidden even
  though your editor handles `text/plain` and can open it.
- `madft info application/x-ipynb+json` reports `default: (none)` even though an
  ancestor (e.g. `application/json` or `text/plain`) has a default — there's no
  **effective default**.
- You can't `set` your editor as the default for those languages without `--force`,
  because the guard requires exact declaration.

## Core concept

One predicate unifies the fix:

> **`handles(app, t)` = `app` declares `t` **or** declares some ancestor of `t`.**

A type is **openable** iff some installed app handles it. Inheritance is now *real*
for visibility, defaults, and `set` — while the **exact** declaration is still tracked
and shown separately (so "what nvim actually declares" vs "what it can open" stays
visible).

## 1. New engine machinery

The engine already has `mimedb` (`ancestor_types(t)` — BFS, nearest-first,
canonical), `appindex` (`apps_for_type`/`declares` — exact), and `defaults`. Add:

- `handles(app_id, t) -> bool` = `appindex.declares(app_id, t) || ancestor_types(t).iter().any(|a| appindex.declares(app_id, a))`.
- `inheritable_apps(t) -> Vec<(AppRef, MimeType)>` = each app that declares some
  ancestor of `t` but does **not** declare `t` itself, paired with the **nearest**
  such ancestor (its "via"). Distinct apps, sorted by id (deterministic).
- `effective_default(t) -> Option<(DesktopId, Option<MimeType>)>`:
  - exact `current_default(t)` set → `(id, None)` (via = exact/blank);
  - else walk `ancestor_types(t)` nearest-first, first ancestor `a` with
    `current_default(a)` → `(id, Some(a))`;
  - else `None`.
- `type_is_inert(t)` (redefined) = NOT openable = `apps_for_type(t).is_empty() &&
  inheritable_apps(t).is_empty()`. (Equivalently: no app handles `t`.)

`effective_default` and `current_default` operate on canonical types; the known
pre-existing alias-keyed-`current_default` bug (logged separately) is out of scope.

## 2. The `default` object (replaces `current_default` in type-centric views)

`TypeInfo` and `LeafType` drop the flat `current_default: Option<String>` in favor of
one structured, provenance-carrying field:

```rust
/// The effective default for a type: who opens it, and via which ancestor (None =
/// the type's own exact default). `None` (the outer Option) = no default at all,
/// exact or inherited.
#[derive(Serialize, Debug)]
pub struct DefaultRef {
    pub app: String,          // desktop id, e.g. "nvim.desktop"
    pub via: Option<String>,  // None = exact; Some("text/plain") = inherited from there
}
// field: pub default: Option<DefaultRef>
```

JSON:
```jsonc
"default": {"app": "eog.desktop",  "via": null}          // exact
"default": {"app": "nvim.desktop", "via": "text/plain"}  // inherited
"default": null                                          // none anywhere
```

This is a **breaking shape change** to `ls --json`/`info --json` (the field renames
from `current_default` to `default` and becomes an object). Acceptable pre-1.0;
explicitly requested. `app <id>`'s `AppTypeRow.current_default` is **unchanged** (that
view is app-centric and exact — added in v0.2.2).

## 3. `info <mimetype>`

`TypeInfo` becomes (changes in **bold**):
- `mime`, `category`, `comment` (unchanged)
- **`default: Option<DefaultRef>`** (was `current_default`)
- `applicable_apps: Vec<AppRef>` — exact declarers (unchanged), `applicable_count` (unchanged)
- **`inheritable_apps: Vec<InheritableApp>`** where `InheritableApp { id, name, via }` — apps that could open/be-set via inheritance, each with the nearest ancestor; **`inheritable_count`**
- `ancestor_types: Vec<String>` (unchanged — the "inherits if unset" chain)

Human:
```
application/x-ipynb+json
  category: Text.Development
  default: code.desktop (via application/json)
  applicable apps: 0
  inheritable apps: 2
    - code.desktop (Visual Studio Code) — via application/json
    - nvim.desktop (Neovim) — via text/plain
  inherits if unset: application/json, text/plain
```
When `default` is exact, the human line omits the `(via …)` suffix. When `default` is
`None`, it prints `(none)`. The `inheritable apps` block is omitted when empty.

## 4. `ls [PATH]`

- **Presence filter** now uses openable: a leaf type shows (without `-a`) iff
  `!type_is_inert(t)` (openable via exact or inherited handler); a subcategory shows
  iff any descendant type is openable. So `Text.Development.Python` and its types
  appear because the editor handles `text/plain`.
- `LeafType` becomes: `mime`, **`default: Option<DefaultRef>`** (was `current_default`),
  `applicable_count` (exact declarers, unchanged), **`inheritable_count`** (new).
- Human leaf row:
  ```
  text/x-python  [default: nvim.desktop (via text/plain), apps: 0, +3 via inherit]
  text/x-csrc    [default: (none), apps: 1]
  ```
  - `default:` shows the effective default + `(via <ancestor>)` when inherited, `(none)` when absent.
  - `apps: N` = exact declarers (unchanged meaning).
  - `, +M via inherit` is appended only when `inheritable_count > 0`.

## 5. `set <app> [target]`

- **Guard relaxed to `handles()` by default.** A candidate is an umbrella type where
  `force || handles_effective(app, t)`, with
  `handles_effective = if opts.exact { declares(t) } else { handles(t) }`.
  Non-candidates go to `skipped_types` (informational); empty candidates →
  `AppHandlesNothingUnderUmbrella` (unchanged).
- **`--exact` flag** (new, in `SetOptions`): restrict to exactly-declared types.
  Inherited-only types are then **not** candidates — for a category umbrella they land
  in `skipped_types` (left blank, still inheriting, writable later by `--no-clobber`);
  for a single inherited-only mimetype target, candidates are empty → guard error.
- **Provenance reporting.** `SetPlan` gains
  `inherited_via: Vec<InheritedSet>` where `InheritedSet { mime, via }`, listing the
  `set_types` entries that matched via inheritance (declared an ancestor, not the type)
  and the nearest ancestor. (A `--force`d type the app doesn't handle at all has no
  entry.) `set_types`/`skipped_types`/`unchanged_types` keep their shapes.
- **Presence filter on the umbrella** (the existing default-hide of inert types) is
  now inheritance-aware via the redefined `type_is_inert`; `--all` still includes
  truly-inert types; an explicit mimetype target or `--types` list still bypasses.

Human `set`:
```
set nvim.desktop as default for Text.Development (9 types):
  + text/x-python  (via text/plain)
  + text/x-go      (via text/plain)
  + text/x-rust    (via text/plain)
  ...
```
(A declared type is listed without a `(via …)` suffix.)

### `--exact` truth table (target inherited, not declared, e.g. nvim + text/css)

| command | result |
|---|---|
| `set nvim text/css` | sets it, `inherited_via: [{text/css, text/plain}]` |
| `set nvim text/css --exact` | error `app-handles-nothing-under-umbrella` |
| `set nvim text/css --exact --force` | writes `text/css` (force overrides) |
| `set nvim text/plain --exact` | sets `text/plain` (declared); other types not in call |
| `set nvim Text.Development --exact` | sets only exact-declared types; inherited-only → `skipped_types`; errors only if nothing declared exactly |

`SetOptions` is now `{ force, no_clobber, exact, show_all, dry_run }`. `--exact` and
`--force` may combine (force wins — sets even un-handled types). `--exact` and
`--no-clobber` combine (exact-declared AND currently-unset).

## 6. Unchanged

- **`get <mimetype>`** — unchanged: bare exact `current_default` (human + `{"default": d}` json). The effective default lives in `info`/`ls`.
- **`apps [target]`** — unchanged: coverage ranked by **exact** declarers (who specifically supports the umbrella). Not inheritance-expanded (out of scope).
- **`app <id>`** — unchanged (v0.2.2): app-centric, `current_default` stays the exact owner.
- `types`, `unset`, `init` — unchanged.

## 7. JSON summary

- **Breaking (shape):** `ls`/`info` `current_default: Option<String>` → `default: Option<DefaultRef {app, via}>`.
- **Additive:** `ls` `LeafType.inheritable_count`; `info` `TypeInfo.inheritable_apps` (+`inheritable_count`); `set` `SetPlan.inherited_via`.
- Unchanged: `get`, `apps`, `app`, error envelope, all other fields.

## 8. Surfaces touched

- `src/engine.rs` — `handles`, `inheritable_apps`, `effective_default`, redefined
  `type_is_inert`; `DefaultRef`/`InheritableApp`/`InheritedSet` structs;
  `TypeInfo`/`LeafType` field swap; `info`/`ls`/`set` rebuilt; `SetOptions.exact`; tests.
- `src/cli.rs` — `human_info`, `human_ls`, `human_set` rendering; `--exact` arg on
  `Set`; thread into `SetOptions`; tests.
- `tests/golden.rs` — inheritance presence, effective default, `set` via inheritance, `--exact`.
- Fixtures — the existing `tests/fixtures/mime/subclasses` already has
  `text/html → text/plain`, `application/xml → text/plain`, `image/svg+xml →
  application/xml`. Add what the new tests need (see §9).
- `Cargo.toml` — `0.3.0`. `README.md` — document inheritance, `default` object, `--exact`.

## 9. Testing

Deterministic, against committed fixtures. The fixture subclass chain gives
`image/svg+xml → application/xml → text/plain`; `eog` declares `image/svg+xml`,
`nvim` declares `text/plain`+`text/html`. Useful cases:
- `handles(nvim, image/svg+xml)` is true (declares `text/plain`, an ancestor); `handles(eog, image/svg+xml)` true (exact).
- `effective_default(image/svg+xml)`: with `text/plain` unset and `image/svg+xml` unset → None; set `text/plain=nvim` (temp config) → `(nvim.desktop, via "text/plain")` for any unset descendant (e.g. `application/xml`).
- `inheritable_apps(application/xml)` includes `nvim` via `text/plain` (nvim declares text/plain, not application/xml); excludes apps that declare application/xml exactly.
- `type_is_inert`: a type with no exact declarer but an ancestor declarer is NOT inert (openable); a type with neither is inert.
- `ls`: a subcategory whose types are only inherited-openable now shows by default; row carries `default` object + `inheritable_count`.
- `set`: `set nvim <inherited-type>` succeeds with `inherited_via` populated; `--exact` on the same → guard error; `--exact --force` writes; category `--exact` skips inherited-only.
- `info`: `default` object shape (exact vs inherited vs none); `inheritable_apps` list with `via`.
- golden: end-to-end CLI for the above incl. `--json` `default` object and `set ... (via …)` human output.

If a richer chain is needed for a clean test (e.g. a JSON/ipynb-like case), add minimal
entries to `tests/fixtures/mime/{types,subclasses}` and the relevant `.desktop`
declarations — keep existing tests green (they assert specific `Other`/category contents).

## 10. Acceptance criteria

- `madft ls Text.Development`-style nodes show types openable via inheritance without `-a`.
- `madft info <inherited-type>` shows the effective `default` with `via`, and an `inheritable apps` list.
- `madft set <editor> <category> [--no-clobber]` sets the inherited languages, no `--force`; the plan reports `(via …)`.
- `--exact` restricts to exact declarations per the truth table.
- `default` is the `{app, via}|null` object in `ls`/`info` (human + json); `get`/`apps`/`app` unchanged.
- `cargo test` + `cargo clippy --all-targets -- -D warnings` pass; no let-chains (MSRV 1.85). `Cargo.toml` = `0.3.0`.

## 11. Out of scope

- `apps` inheritance-expansion (coverage stays exact).
- `get` effective default (stays exact by user choice).
- The pre-existing alias-keyed `current_default` bug (logged; separate fix).
- Remote category DB and other long-deferred seams.
