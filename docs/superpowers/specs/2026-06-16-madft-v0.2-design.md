# madft v0.2.0 — root targets, app-first `set`, `--no-clobber`, and `ls` cleanup

Status: approved design. Builds on the MVP (`2026-06-15-madft-design.md`) and the
post-MVP addendum (`2026-06-16-madft-info-category-and-app-query-design.md`).

This release makes the CLI usable end-to-end *without* a front-end or a pile of
piped calls. The headline is the app-first `set` signature plus `--no-clobber`,
which together cover the "make this app my default for everything it can handle
that I haven't already decided" workflow in one command.

## Why a minor bump

The `set` positional order changes (`set <target> <app>` → `set <app> [target]`).
That is a breaking change to the command-line surface, so the package version goes
`0.1.0` → `0.2.0`. The `--json` schema changes are all *additive* (new fields, new
optional positionals), so the machine contract stays backward-compatible.

Human (non-JSON) output is **not** a contract; the `ls` slash removal and the
header rendering are free to change.

## 1. `ls` human output

Two changes, both human-only (JSON `LsResult` is unchanged: `subcategories` stays a
list of dotted paths, `types` stays a list of `LeafType`).

### 1.1 Drop the trailing slash on categories

Today `human_ls` prints `Media.Audio/`. Remove the `/`. Rationale: a mimetype
always contains `/` and a category path never does, so the trailing slash both is
redundant *and* visually collides with the slash inside `video/mp4`. After the
change the rule is crisp:

> A line containing `/` is a mimetype; a line without `/` is a category.

### 1.2 Section headers only when a node has both groups

Interior nodes can own types (the arena puts `types` on every node), so a single
`ls` can show both subcategories and direct types. The model already supports this;
the default tree just doesn't exercise it yet.

Rendering rule:

- Node has **both** subcategories and direct types → print a `categories:` header
  over the (indented) category list, then a `types:` header over the (indented)
  type list.
- Node has **only one** group → list it bare (no lone header), preserving the clean
  common case (this includes root `ls`, which only ever has categories).

```
$ madft ls Media            # both groups present
categories:
  Media.Audio
  Media.Video
types:
  application/ogg  [default: (none), apps: 0]

$ madft ls Media.Video      # only types
video/mp4  [default: mpv.desktop, apps: 3]
video/x-matroska  [default: (none), apps: 1]

$ madft ls                  # only categories (root)
Media
Other
Web
```

Indentation under a header is two spaces. The per-type annotation
(`[default: …, apps: N]`) is unchanged.

## 2. Root targets and the app-first `set`

### 2.1 Optional target, `.` as the root token

`apps` and `set` gain an optional target that defaults to **root** (the whole tree).
A literal `.` is also accepted as an explicit root token, for filesystem muscle
memory and scripts. Root's display label (the `target` field in JSON and the human
output) is `.`.

- `Command::Apps { target: Option<String> }`
- `Command::Set { app: String, target: Option<String>, types, force, no_clobber, dry_run }`

Resolution (`resolve_umbrella`) takes an optional target:

- `None` or `"."` → root: umbrella = DFS union of `types_under(root)` over
  `tree.roots()` in arena order. Because every placed type lives under exactly one
  root and `Other` is total, this is exactly the set of all placed types, with no
  cross-node de-duplication needed. Label = `"."`.
- contains `/` → a mimetype (umbrella = just that canonical type). Unchanged.
- otherwise → a category path via `node_by_path`. Unchanged.

A small helper on `CategoryTree` returns the root-union in DFS order (e.g.
`all_types()`), so the engine doesn't reach into arena internals.

### 2.2 `set` becomes app-first

`set <app> [target]`:

```
madft set mpv               # everything mpv declares, system-wide (root)
madft set mpv Media         # scoped to a category umbrella
madft set mpv video/mp4     # a single type
madft set mpv .             # explicit root, same as omitting the target
```

The app is the required first positional; the target is the optional second. This
mirrors `apps [target]` and reads naturally with the app-centric flags
(`--force`, `--no-clobber`). `unset <mimetype>` is unchanged (still type-only).

`apps [target]`:

```
madft apps                  # all apps, ranked by total coverage (root)
madft apps Media            # apps covering the Media umbrella
madft apps video/mp4        # apps declaring one type
```

## 3. `--no-clobber` (alias `--if-unset`)

A flag on `set` that restricts the write to umbrella types that currently have **no
default**. Primary spelling `--no-clobber` (cp/mv precedent: don't overwrite an
existing value); `--if-unset` is a clap `visible_alias`, so both appear in `--help`
and both work. No short flag.

Semantics, computed against the resolved umbrella:

1. **Candidates** = umbrella types that pass the `--types` filter (if any) and are
   either declared by the app or `--force`d. This is the existing partition: types
   outside the filter are ignored; declared/forced types are candidates; the rest go
   to `skipped_types` (undeclared, informational — unchanged meaning).
2. **Empty-candidate guard** (runs *before* the no-clobber split): if there are no
   candidates, the app genuinely handles nothing under the umbrella → error
   `AppHandlesNothingUnderUmbrella`. This is true with or without `--no-clobber`: an
   app that declares nothing here is a failed call regardless of the flag.
3. **No-clobber split** (only when the flag is set): partition candidates by current
   default —
   - no current default → `set_types` (will be / was written),
   - has *any* current default → `unchanged_types` (kept, not written).
   A type already set to a *different* app is kept (that's the whole point); a type
   already set to *this* app is also "kept" (it already has a default). Under the
   flag, `set` only fills blanks.
4. **Success with nothing to do**: if `--no-clobber` leaves `set_types` empty
   because every candidate already had a default, that is **success** (exit 0,
   `written: false`), *not* the empty-candidate error. The error in step 2 is about
   "this app can't handle anything here"; this case is "everything's already
   decided." They are distinct and must not be conflated.

Without `--no-clobber`, behavior is unchanged: `set_types` = all candidates,
`unchanged_types` is empty, and an idempotent re-set still writes nothing.

`--no-clobber` composes with `--force` (fill blanks even for undeclared types) and
`--dry-run` (preview the split without writing).

### `SetPlan` additions (additive JSON)

```rust
pub struct SetPlan {
    pub app: String,
    pub target: String,           // "." for root
    pub set_types: Vec<String>,
    pub skipped_types: Vec<String>,   // undeclared (unchanged meaning)
    pub unchanged_types: Vec<String>, // NEW: kept by --no-clobber (had a default)
    pub forced: bool,
    pub no_clobber: bool,             // NEW
    pub dry_run: bool,
    pub written: bool,
}
```

Human rendering gains a line when `unchanged_types` is non-empty, e.g.
`kept (already set): image/png, image/gif`.

## 4. Category-name grammar — colon stays reserved

No change to behavior; recorded here so it isn't revisited. Category names remain
`[A-Za-z0-9 _-]` — `.` (path delimiter), `/` (mimetype delimiter), and `:` are all
disallowed. `:` is kept reserved even though no current syntax uses it, so a future
type-addressing form like `Super.Sub:mime/type` can be added without a breaking
change. The validator already rejects `:`; this is a no-op for the implementation.

## 5. Out of scope (still deferred)

No dedicated "list unset" command: the query side stays greppable
(`madft ls Media` prints `[default: (none)]`; `--json | jq` for scripts). The
`--no-clobber` flag covers the *mutation* side of the unset-aware workflows. Also
unchanged from prior specs: `RemoteSource`, TUI wiring, fuzzy app matching,
`[Added]`/`[Removed]` management, multi-app fallback lists, file locking.

## 7. Acceptance criteria

- `madft ls` and `madft ls <leaf>` show no trailing slashes; a node with both
  groups shows `categories:`/`types:` headers; single-group nodes render bare.
- `madft set mpv` sets mpv as default for every type it declares system-wide;
  `madft set mpv Media` scopes it; `madft set mpv video/mp4` sets one type;
  `madft set mpv .` equals the no-target form.
- `madft apps` lists all apps ranked by coverage over the whole tree; `madft apps .`
  is identical; `madft apps Media` is scoped.
- `madft set mpv --no-clobber` writes only types mpv declares that are currently
  unset; already-set types appear in `unchanged_types`/the kept line; an all-set
  result exits 0 with `written: false`; an app that declares nothing under the
  umbrella still errors `AppHandlesNothingUnderUmbrella`.
- `--no-clobber` and `--if-unset` are interchangeable.
- JSON schemas are additive: existing fields keep their meaning; `unchanged_types`
  and `no_clobber` are new; `target` is `.` for root.
- `cargo test` and `cargo clippy --all-targets -- -D warnings` pass. No let-chains
  (MSRV 1.85).
