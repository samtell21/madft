# madft v0.2.2 — `app` shows types it's the default for but doesn't declare

Status: approved design (pending spec review). Bug fix + additive field. Builds on
v0.2.1.

## Problem

`madft app <id>` lists only the app's **declared** `MimeType=` entries. A type the
app is the current **default** for but does not declare is invisible, and the
`default for N` count undercounts. Two real cases:

- `madft set nvim text/css --force` sets nvim as default for `text/css`, which nvim
  doesn't declare → `madft app nvim` shows neither the row nor the default count.
- `org.pwmt.zathura.desktop` is the effective default for `application/pdf`, but the
  *declaring* file is the separate `org.pwmt.zathura-pdf-mupdf.desktop` →
  `madft app org.pwmt.zathura` reports "declares 0 types, default for 0" even though
  `madft info application/pdf` shows zathura as the default.

`info` is already correct — it lists *declaring* apps, and the default app
legitimately need not be among them. The bug is confined to `app`.

## Fix

The `app` row set becomes **`declared ∪ {types this app is the effective default for}`**,
with a new `declares: bool` per row distinguishing the two.

### 1. `Defaults` reverse lookup

`Defaults` currently answers only per-type (`current_default(t)`). Add:

```rust
/// The effective default per type across the precedence chain (highest file
/// wins per type), folded into one map. Keys are as-written in mimeapps.list
/// (callers canonicalize). Deterministic order.
pub fn effective_defaults(&self) -> BTreeMap<MimeType, DesktopId>
```

Fold `self.files` highest-precedence-first, inserting each `(type, id)` only if the
type is not already present (first file wins — same rule as `current_default`).

### 2. `Engine::app(id)`

- Build a **canonical effective map** once: fold `defaults.effective_defaults()`,
  canonicalizing each key via `mimedb` (first wins on canonical collision). Use it
  for both the reverse lookup and per-row default detection, so alias-keyed defaults
  are handled consistently.
- `declared` = canonical, deduped `app.mimetypes` (as today).
- `default_for_types` = canonical keys of the effective map whose value == `app_id`.
- **Row set** = `declared ∪ default_for_types`, deduped, ordered **default-first then
  mime** (unchanged ordering).
- Each `AppTypeRow` gains `declares: bool` — `true` iff the type is in `declared`.
- `is_default` = canonical-effective default of the row's type == `app_id`
  (so every `default_for_types` row is `is_default: true`; a `declares==false` row is
  therefore always `is_default==true`).
- `current_default` = the canonical-effective default of the row's type (`Option`).
- `AppReport.declares` (count) = rows where `declares == true` (= the app's real
  `MimeType=` count; meaning unchanged). `AppReport.default_for` = rows where
  `is_default == true` — now accurate (includes undeclared defaults).

```rust
#[derive(serde::Serialize, Debug)]
pub struct AppTypeRow {
    pub mime: String,
    pub category: Option<String>,
    pub declares: bool,        // NEW
    pub is_default: bool,
    pub current_default: Option<String>,
}
```

### 3. Human rendering (`human_app`)

A fixed leading tag column: `DEFAULT` on `is_default` rows, blank otherwise (chosen
for greppability — `madft app nvim | grep DEFAULT` lists the default-for rows; the
uppercase tag won't collide with the lowercase `default for` header). A trailing
note conveys the row's status:

- `declares == false` → `  (not declared)` (always an `is_default` row).
- `declares == true && !is_default && current_default.is_some()` → `  (default: <id>)`
  (shows which *other* app currently owns a type this app declares).
- otherwise → no note.

```
nvim.desktop (Neovim) — declares 2 types, default for 1:
  DEFAULT  text/css  [Web]  (not declared)
           text/plain  [Documents]

org.pwmt.zathura.desktop (Zathura) — declares 0 types, default for 1:
  DEFAULT  application/pdf  [Documents]  (not declared)
```

Category falls back to `—` when `None` (as today). Rows are not column-padded
(consistent with the rest of the tool's human output).

### 4. JSON

`declares` is added to each `AppTypeRow` (additive); the `types` array now also
carries undeclared-default rows. Schema stable, content fuller.

```jsonc
// madft app nvim --json   (after: madft set nvim text/css --force)
{
  "id": "nvim.desktop",
  "name": "Neovim",
  "declares": 2,
  "default_for": 1,
  "types": [
    {"mime": "text/css",   "category": "Web",       "declares": false, "is_default": true,  "current_default": "nvim.desktop"},
    {"mime": "text/plain", "category": "Documents", "declares": true,  "is_default": false, "current_default": null}
  ]
}
```

## 5. Surfaces touched

- `src/defaults.rs` — `effective_defaults()` + unit test; import `BTreeMap`.
- `src/engine.rs` — `AppTypeRow.declares`; rewrite `app` to union + canonical
  effective map; engine tests.
- `src/cli.rs` — `human_app` (DEFAULT tag + notes); cli test.
- `tests/golden.rs` — a golden where an app is default for an undeclared type.
- `Cargo.toml` — `version = "0.2.2"`.
- `README.md` — note that `app` lists both declared types and types it's the default
  for (with the `declares` flag); refresh the `app` `--json` example.

## 6. Testing

- `defaults.rs`: `effective_defaults` folds the chain (highest wins per type),
  returns every defaulted type once.
- `engine.rs`: an engine whose mimeapps sets `image/png=mpv.desktop` (mpv does NOT
  declare `image/png`) → `app("mpv")` includes an `image/png` row with
  `declares=false, is_default=true`; `default_for` counts it; declared types still
  present with `declares=true`. The existing `app_reports_declared_types_defaults_and_categories`
  (read-only fixture, no undeclared defaults) stays green with `declares=true` on its rows.
- `cli.rs`: `human_app` shows the `DEFAULT` tag and `(not declared)` for an
  undeclared default row.
- `tests/golden.rs`: build an engine over a temp config containing
  `image/png=mpv.desktop`; `app mpv --json` → the `image/png` row has
  `declares=false`, `is_default=true`.

## 7. Acceptance criteria

- `madft app <id>` lists every type the app is the current default for, even
  undeclared ones; `default for N` matches `info`'s view of who's default.
- Each row carries `declares` (json) / a `DEFAULT` tag + `(not declared)` note (human).
- `madft app nvim | grep DEFAULT` lists exactly the types nvim is the default for.
- Declared-but-not-default rows still show, with the owning app in `(default: <id>)`.
- JSON is additive (new `declares` field; `is_default`/`current_default`/counts
  unchanged in meaning); no other command changes.
- `cargo test` + `cargo clippy --all-targets -- -D warnings` pass; no let-chains
  (MSRV 1.85). `Cargo.toml` is `0.2.2`.

## 8. Out of scope

- No change to `info`, `ls`, `types`, `apps`, `set`, `get`, `unset`, `init`.
- No reverse "what could this app be forced onto" enumeration (only what it currently
  *is* the default for).
- The presence filter (`-a/--all`) still does not affect `app`.
