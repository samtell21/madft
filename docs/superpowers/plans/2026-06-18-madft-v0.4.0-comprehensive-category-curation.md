# madft v0.4.0 — Comprehensive Category Curation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pull every desktop-relevant mimetype out of the flat `Other` node into curated categories, and first-class the libmagic-only sniff types (the `font/sfnt` class) so madft works out of the box for content-sniffing openers.

**Architecture:** Pure data curation of `data/categories.toml`. No Rust changes — the engine already first-classes any type listed in the tree (proven: `source.rs` does not validate against the universe, `merge.rs` places listed types unconditionally). The user's live `~/.local/share/madft/categories.toml` is byte-identical to the built-in and is regenerated with `madft init --force` at the end.

**Tech Stack:** Rust 1.85+ (edition 2024), TOML category grammar, `cargo test`, shared-mime-info (`/usr/share/mime`), libmagic (`/usr/share/misc/magic`).

## Global Constraints

- **No engine/Rust changes.** Data-only edits to `data/categories.toml`. (Code edits are an automatic task failure.)
- **Inclusion rubric (verbatim):** a type earns a category **iff** a user could plausibly want a **default handler app (terminal *or* GUI)** to open it, **and** it is a recognizable real-world format. Terminal apps count (`nvim.desktop` is `Terminal=true`).
- **Excluded → stays in `Other`:** scientific/instrument formats; raw executables & byte-code; transport/internal/protocol types; parameterized types (any value containing `;`); odd-casing artifacts (e.g. `application/CDFV2`); legacy aliases that canonicalize onto an already-placed type.
- **Single placement invariant:** every type appears in **exactly one** category's `types` array across the whole file. The TOML parser enforces this (`DuplicatePlacement`); `cargo test default_categories_is_valid` is the gate.
- **Category names:** match existing charset rules (alphanumerics; dotted paths quoted, e.g. `["Media.Video"]`). Reuse existing categories before inventing new ones.
- **Version:** bump `Cargo.toml` to `0.4.0` in the final task only.
- **Canonicalize before listing:** if unsure whether a type is an alias, check `grep '^<type> ' /usr/share/mime/aliases`; list the canonical form, never the alias.

## Reference commands (used throughout)

Build the binary once so the inspection commands work:

```bash
cargo build --release
MADFT=./target/release/madft
```

Generate the two candidate inventories (regenerate any time):

```bash
# A) shared-mime-info types currently stranded in Other (760 today)
$MADFT types Other -a | sort > /tmp/madft-other.txt

# B) libmagic-only types (emitted by `file`, absent from shared-mime-info) —
#    the font/sfnt class. Drop parameterized (`;`) entries.
comm -23 \
  <(grep -rhoE '!:mime[[:space:]]+\S+' /usr/share/misc/magic* | awk '{print $2}' | sort -u) \
  <(sort /usr/share/mime/types) \
  | grep -v ';' > /tmp/madft-libmagic-only.txt
```

Per-domain candidate slice (example for fonts):

```bash
grep -iE 'font' /tmp/madft-other.txt /tmp/madft-libmagic-only.txt
```

Duplicate-placement / validity gate (run after every edit):

```bash
cargo test --lib default_categories_is_valid -- --nocapture
```

Other-shrinkage tracker:

```bash
cargo build --release -q && $MADFT types Other -a | wc -l   # 760 at start
```
